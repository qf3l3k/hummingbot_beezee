Beezee connector resilience and restart recovery release.

Included:

- Safe recovery of Beezee native order ids after a Hummingbot restart
- Timestamp-based disambiguation for otherwise identical open orders
- Protection against duplicate order-created events after confirmed transactions
- Improved transaction and unresolved-order diagnostics
- Safe handling of empty order books and non-finite prices
- Regression tests covering the new recovery and pricing behavior

Validated path:

- Beezee testnet
- Private-key wallet mode
- Limit-order create, cancel, fill detection, and restart recovery

Notes:

- Pure market making requires a reference price. An entirely empty Beezee book has no mid-price, so use a custom/external price source or seed liquidity before running PMM.
- Mnemonic wallet mode remains unvalidated for trading and should be tested only with a disposable wallet.
