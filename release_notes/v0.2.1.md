Hardening release that brought the Beezee connector to a working Beezee testnet trading state.

Included:
- Fixed dynamic REST throttler ids for Beezee-specific endpoints
- Added denom metadata URL encoding and fallback handling for IBC and factory denoms
- Added native `ubze` fallback metadata and reduced repeated metadata warning spam
- Added Beezee server time helper required by Hummingbot polling
- Fixed encrypted mnemonic/private-key config loading on restart
- Passed read-only address through Hummingbot connect and balance flows
- Fixed Beezee signer address hashing to match Beezee account format
- Added signer/configured address validation and wallet startup logging
- Relaxed readiness gating for Beezee polling-based private state and symbol mapping startup
- Added readiness diagnostics for pending connector flags
- Fixed Cosmos account sequence handling for concurrent order and cancel transactions
- Improved cancel flow by resolving Beezee exchange order ids before broadcast

Validated on testnet:
- balances
- order book initialization
- order creation
- cancel submission

Notes:
- Private-key wallet mode is the validated path for testnet trading in this release
- Mnemonic mode still needs separate derivation validation before being treated as production-safe
