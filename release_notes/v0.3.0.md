Beezee order lifecycle safety and resilience release.

Included:
- Cancel requests remain pending until the cancel transaction is confirmed on-chain
- Conservative exchange order-id correlation that excludes orders present before submission and rejects ambiguous candidates
- Fill attribution ignores historical trades that predate the relevant active order bucket
- Block websocket disconnect recovery through polling and reconnect attempts
- Create and cancel transaction hashes logged with client and exchange order identifiers
- Regression tests for cancel confirmation and ambiguous order matching

Validated path:
- Beezee testnet
- private-key wallet mode

Notes:
- This release improves live order safety and observability.
- Mnemonic wallet mode remains unvalidated and should not be used for trading.
